# 并行冲突处理 Playbook（Lead）

## 1. 冲突等级
- L1: 同文件不同区域（可直接合并）
- L2: 同文件相邻区域（需要 owner 协调）
- L3: 合同/Schema 冲突（必须 Lead 决策）

## 2. 处理流程
1. 提交冲突摘要（文件、分支、目标行为）
2. 对照 `api-contract-v1` 判断是否破坏合同
3. 按优先级决策：
   - 合同正确性
   - 数据一致性
   - UI 兼容性
4. 冲突解决后跑最小验证集：
   - API tests
   - UI tests
   - 相关 E2E

## 3. 禁止项
- 未经审批修改 frozen 合同
- 以“先能跑”为理由跳过测试
- 冲突解决后不回归验证

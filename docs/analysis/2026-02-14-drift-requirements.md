# Drift 需求分析（MVP -> Pro）

## 1. 产品定位
Drift 是一个轻量级“个人稳定度早期预警系统”，面向现代职场人（重点是远程工作者与高压从业者）。
它不做心理诊断，不替代医疗建议，不做复杂生产力管理。它只做一件事：
通过每天几秒钟的低负担打卡，识别“缓慢下滑”的趋势，并在问题变严重前提醒用户。

## 2. 目标用户与核心场景
- 远程工作者：社交隔离、作息失衡、工作边界模糊。
- 高压岗位从业者：长期高压、恢复不足、忽略早期风险信号。
- 自我管理意识强但不愿重度记录的人：需要“极简、持续、可见趋势”。

关键场景：
- 每天固定时段快速打卡（<= 10 秒）。
- 一周/一月回看趋势，识别慢性下滑。
- 收到温和提醒后，做低成本调整（休息、联系重要关系、降低负荷）。

## 3. 产品目标与边界
### 3.1 产品目标
- 提高用户对“慢性失衡”的可见度。
- 提供可解释的趋势提醒，而不是泛泛打分。
- 提升日打卡持续率与“提醒后行动率”。

### 3.2 非目标（Out of Scope）
- 心理疾病诊断、治疗建议、危机干预替代。
- 企业绩效追踪、工时监管。
- 重型任务管理、项目管理。

## 4. 功能需求
### 4.1 MVP 功能
- 每日一次打卡（字段建议）：
  - 精力（1-5）
  - 压力（1-5）
  - 社交互动感（1-5）
  - 与重要联系人互动（是/否，可选）
- 打卡提醒：
  - 可设置每日提醒时间与时区
  - 漏打卡补录（当日）
- 趋势展示：
  - 7 天、30 天趋势图
  - 指标单独趋势 + 综合漂移指数（Drift Index）
- 漂移检测（规则版）：
  - 连续低精力、连续高压力、社交持续下滑
  - 满足阈值触发“温和提醒”
- 提醒与建议（模板版）：
  - 简短、非命令式、可执行建议（2-3 条）
- 账号与数据：
  - Email/社交登录
  - 数据可导出（CSV）

### 4.2 Pro 功能（后续）
- AI 个性化趋势解读（周报/月报）
- AI 行动建议（基于个人历史）
- 风险解释增强（“为何触发”可解释因子）
- 自定义维度与权重

## 5. 非功能需求
- 体验：打卡路径极短，移动端单手完成。
- 性能：关键页面 P95 < 1.5s（良好网络），打卡提交 < 400ms（服务端处理）。
- 稳定性：核心 API 可用性 >= 99.9%。
- 安全与隐私：
  - 传输加密（HTTPS）
  - 敏感数据最小化收集
  - 支持数据删除请求
- 合规文案：明确“非医疗用途”。

## 6. 风险与约束
- 误报/漏报：阈值过敏或过钝。
- 用户负担回升：字段变多导致留存下降。
- 解释不清：提醒若缺乏可解释性会降低信任。

## 7. 核心指标（北极星与配套）
- 北极星：连续 28 天打卡完成率。
- 配套指标：
  - D1/D7/D30 留存
  - 提醒触发率与提醒后 48h 行动率
  - 漏打卡率
  - 用户主观“提醒有帮助”评分

## 8. 验收标准（MVP）
- 用户可在 10 秒内完成打卡。
- 系统可在 7 天内生成趋势，并在符合规则时提醒。
- 提醒内容可解释触发原因（至少 1 条具体原因）。
- 用户可查看历史趋势并导出数据。

## 9. 待确认决策
- 首发平台优先级：Web PWA 或 iOS/Android。
- 提醒通道：仅应用内/邮件/推送。
- 免费版保留数据时长与 Pro 定价。
